# WebWaka AI Platform (`webwaka-ai-platform`) QA Certification

**Prepared by:** Manus AI
**Date:** April 2026
**Target Repository:** `webwaka-ai-platform`

## 1. Audit Scope

This QA certification covers the implementation of PII redaction middleware, semantic caching, and dynamic model routing in `webwaka-ai-platform`.

## 2. Acceptance Criteria

| ID | Feature | Acceptance Criteria | Status |
| :--- | :--- | :--- | :--- |
| QA-AIP-1 | PII Redaction | `src/middleware/redaction.ts` successfully redacts Nigerian phone numbers, BVNs, and emails from prompts before sending to the LLM. | PENDING |
| QA-AIP-2 | Semantic Caching | `src/services/cache.ts` correctly queries Cloudflare Vectorize and returns cached responses for prompts with >0.95 similarity. | PENDING |
| QA-AIP-3 | Dynamic Routing | Prompts shorter than 50 words are routed to Cloudflare AI (Llama-3) by default, while longer prompts use OpenRouter (GPT-4o). | PENDING |
| QA-AIP-4 | Unit Tests | All new middleware and services have passing unit tests in `src/**/*.test.ts`. | PENDING |

## 3. Offline Resilience Testing

- The AI platform is a backend service; offline resilience applies to its clients (the vertical suites).
- However, the platform must gracefully handle upstream provider outages (e.g., OpenRouter 503s) by automatically falling back to Cloudflare AI.

## 4. Security & RBAC Validation

- Verify that the PII redaction middleware cannot be bypassed by malformed JSON payloads.
- Ensure that cached responses are strictly isolated by `tenantId` (Tenant A cannot receive a cached response generated by Tenant B).

## 5. Regression Guards

- Run `npm run test` to ensure 100% pass rate.
- Run `npm run build` to ensure no TypeScript compilation errors.
- Verify that the `ai.usage.recorded` event is still emitted correctly, even when a response is served from the semantic cache (to track cache hit savings).
